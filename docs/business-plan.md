# 일해요 (1haeyo) — 사업 기획서

> 초단기 일자리 실시간 매칭 플랫폼
> **"원하면, 바로 일해요 — 1haeyo"**

### 브랜드 네이밍

**1haeyo**는 두 가지로 읽힌다:

- **원해요** — 구인자는 사람을 원하고, 구직자는 일을 원한다
- **일해요** — 원하면, 바로 일한다

숫자 1이 "일(work)"과 "원(want)"을 동시에 담고 있다.

> 슬로건: **"원하는 순간, 바로 일해요"**

---

## 1. 문제 정의

### 1-1. 노인은 소비자가 아니라 생산자다

기존 실버 산업은 노인을 돌봄과 복지의 **소비자**로만 바라봤다. 하지만 현실은 다르다:

- 현대 시니어는 육체적으로 젊고, 정신도 젊고, 경험도 많고, 생산성도 높다
- 노인네 취급받고 싶어하지 않는다
- 심심하고, 돈도 부족하고, 사회적으로 연결되고 싶어한다
- **일하고 싶어한다**

### 1-2. 양쪽 모두 문제가 있다

**구직자 측 (일하고 싶은 사람)**
- 연금만으로는 생활이 안 된다 (월평균 연금 86만원, 최소 생활비 136만원의 63%)
- 매일 출근하는 정규직은 어렵지만, 가끔 일하고 싶다
- 면접·이력서 과정이 부담스럽다
- 사회적 고립감을 느낀다

**구인자 측 (사람이 필요한 곳)**
- 서비스업 인력 부족이 만성적이다 (숙박/음식점업 인력 부족 5.6만 명)
- 알바 노쇼, 갑작스런 공석에 대응이 어렵다
- 짧은 시간만 사람이 필요한데 정규 채용은 부담이다
- 급하게 사람을 구할 방법이 마땅치 않다

## 3. 솔루션 — 일해요 (1haeyo)

### 3-1. 한 줄 정의

> **"면접 없이, 이력서 없이, 우버처럼 즉시 매칭되는 초단기 일자리 플랫폼"**

### 3-2. 핵심 구조

```
구인자                              워커
"오늘 12시 홀서빙 1명 급구"          "지금 일할 수 있어요"
        ↓                              ↓
    [일감 등록]                    [대기 상태]
        ↓                              ↓
    ┌──────────── 일해요 플랫폼 ────────────┐
    │  GPS 기반 반경 매칭                      │
    │  워커 경력/평점 기반 추천                  │
    │  푸시 알림 → 수락 → 확정                 │
    └──────────────────────────────────────┘
        ↓                              ↓
    [매칭 완료]                    [출발]
        ↓                              ↓
    [위치 인증 확인]                [현장 도착 → 체크인]
        ↓                              ↓
    [근무 완료 확인]                [체크아웃]
        ↓                              ↓
    [양방향 평점/리뷰]              [양방향 평점/리뷰]
        ↓                              ↓
    [비용 정산]                    [급여 수령]
```

### 3-3. 핵심 원칙

1. **하이퍼로컬**: GPS 기반, 내 주변 일감만 노출
2. **실시간 매칭**: 구인 등록 즉시 반경 내 워커에게 푸시 알림. 5분 안에 매칭
3. **위치 인증**: 출퇴근 시 현장 GPS 체크인/체크아웃
4. **이력서 불필요, 경력은 축적**: 면접·이력서 없이 바로 투입. 단, 플랫폼 내에서 근무 이력·경력·평점이 자연스럽게 쌓인다
5. **양방향 신뢰 시스템**: 카카오택시·우버처럼 구인자↔워커 양방향 별점/리뷰
6. **즉시 정산**: 근무 완료 후 빠른 정산

## 4. 신뢰 시스템 — 이력서를 대체하는 구조

전통적 채용은 이력서와 면접으로 신뢰를 확보한다. 일해요는 **플랫폼 내 축적 데이터**로 이를 대체한다.

### 4-1. 워커 프로필에 쌓이는 것들

| 항목 | 설명 |
|---|---|
| **종합 별점** | 구인자가 매 근무 후 평가 (5점 만점) |
| **근무 이력** | 업종별 근무 횟수 (예: 식당 32회, 물류 15회, 사무보조 8회) |
| **출석률** | 수락 후 실제 출근한 비율 |
| **완료율** | 근무를 끝까지 완수한 비율 |
| **재호출률** | 같은 구인자가 다시 부른 비율 (숨은 핵심 지표) |
| **경력 태그** | 자동 부여: "식당 베테랑", "물류 숙련자" 등 |
| **우수 워커 배지** | 일정 기준 충족 시 부여 (별점 4.8+, 출석률 95%+ 등) |

### 4-2. 구인자 프로필에 쌓이는 것들

| 항목 | 설명 |
|---|---|
| **종합 별점** | 워커가 매 근무 후 평가 |
| **정산 속도** | 약속된 시간에 정산했는지 |
| **근무 환경 리뷰** | 워커들의 현장 후기 |
| **재지원율** | 워커가 다시 일하고 싶어하는 비율 |

### 4-3. 매칭 우선순위

```
1순위: 해당 업종 경험 있는 워커 (경력 태그 매칭)
2순위: 별점 높은 워커
3순위: 출석률/완료율 높은 워커
4순위: 거리가 가까운 워커
5순위: 선착순
```

→ **이력서 없이 시작하되, 시간이 지나면 플랫폼 경력이 곧 이력서가 된다.**
→ 카카오택시에서 평점 높은 기사님을 선호하듯, 일해요에서도 평점 높은 워커를 구인자가 선호하게 된다.

## 5. 타겟 유저

### 5-1. 워커 (일하는 사람)

| 세그먼트 | 특징 | 예상 비율 | 핵심 니즈 |
|---|---|---|---|
| **시니어 (55~75세)** | 메인 타겟. 시간 많고, 경험 풍부 | 40~50% | 용돈 + 사회참여 + 의미있는 삶 |
| **청년 백수/구직자** | 취업 전 사이사이, 일당 선호 | 20~25% | 즉시 현금 + 경험 |
| **직장인 부업** | 퇴근 후, 주말, 세컨잡 | 15~20% | 추가 수입 |
| **프리랜서/긱워커** | 유연한 일정, N잡러 | 10~15% | 유연성 |

### 5-2. 구인자 (일감을 올리는 사업체)

**Tier 1 — MVP 단계 핵심 타겟**

| 업종 | 일해요가 필요한 상황 | 예상 일감 |
|---|---|---|
| **식당/카페** | 알바 노쇼, 점심 피크, 단체 예약, 주말 | 홀서빙, 설거지, 주방보조, 카운터 |
| **편의점** | 야간/주말 대타, 갑작스런 공석 | 카운터, 진열, 재고 정리 |
| **물류/배송** | 물량 폭주 시 임시 인력 | 택배 분류, 창고 정리, 상하차 |
| **행사/이벤트** | 당일만 필요한 스태프 | 안내, 세팅, 정리, 주차 안내 |

**Tier 2 — 확장 단계**

| 업종 | 예상 일감 |
|---|---|
| **부동산** | 매물 안내 동행, 현장 사진 촬영, 오픈하우스 보조 |
| **법무/세무/회계** | 서류 정리, 데이터 입력, 고객 접수 보조 |
| **학원/교육** | 셔틀 운행, 학생 관리 보조, 시험 감독 |
| **병원/약국** | 접수 보조, 원무과 대기 안내 |
| **매장/소매** | 재고 실사, 시즌 세일 인력, 오픈/마감 |

**Tier 3 — 시니어 고급 인력 활용**

| 업종 | 예상 일감 |
|---|---|
| **컨설팅/멘토링** | 창업 상담, 경영 자문 (은퇴 전문직) |
| **번역/통역** | 단발성 번역, 외국인 고객 응대 |
| **IT 보조** | 시니어 대상 스마트폰/앱 교육 |
| **아파트/관리** | 시설 점검 보조, 입주민 민원 대응 |

## 7. 경쟁 환경

### 7-1. 직접 경쟁자

| 서비스 | 특징 | 일해요와의 차이 |
|---|---|---|
| **급구** | 단기 알바 매칭, AI 추천, 근태관리+정산 | 시니어 특화 없음, 매칭 속도 차이 |
| **알바프리** | 하루 단위 초단기 알바 | 소규모, 신뢰 시스템 부재 |
| **초단알바** | 위치기반, AI 매칭, 중장년 포함 | 중장년 키워드만 있을 뿐 특화 아님 |
| **뉴워커** | 긱워커 플랫폼 | 젊은층 중심 |

### 7-2. 잠재적 경쟁자 — 당근마켓

당근마켓은 **가장 위협적인 잠재 경쟁자**다.

| | 당근마켓 | 일해요 |
|---|---|---|
| **본질** | 동네 커뮤니티/중고거래 (알바는 부가기능) | 초단기 노동 매칭이 **본업** |
| **매칭 방식** | 게시판형 (올리고 연락 기다림) | 호출형 (우버처럼 즉시 매칭) |
| **매칭 속도** | 수 시간~수 일 | **5분 이내 목표** |
| **정산** | 직거래 (플랫폼 개입 없음) | 플랫폼이 중간 정산 |
| **신뢰 시스템** | 매너 온도 (범용) | 근무 특화 (별점, 출석률, 경력) |
| **시니어 친화** | 특별히 없음 | **핵심 설계 원칙** |
| **강점** | 거대한 사용자 풀, 동네 신뢰 네트워크 | 속도, 전문성, 정산, 신뢰 |

**일해요의 대응 전략:**
- 당근이 "동네 만물상"이라면, 일해요는 **"동네 인력 119"**
- 당근이 할 수 없는 것: 즉시 호출, 전문 신뢰 시스템, 근로계약/정산 자동화
- 당근이 알바 기능을 강화하더라도, 초단기 노동에 올인한 전문 플랫폼의 깊이를 따라오기 어렵다

| **상권 밀집** | 미금역, 야탑역, 서현역 중심 대형 상권 |
| **업종 다양성** | 식당, 카페, 편의점 + 법무법인, 회계사, 세무사, 부동산, 당구장, 만화방 등 |
| **교통** | 지하철역 중심 도보 생활권 → 하이퍼로컬에 최적 |
| **사업체 수** | 성남시 전체 66,333개 (숙박/음식 11,429개, 도소매 16,123개) |

### 8-2. 확장 로드맵

```
Phase 1: 분당 (미금·야탑·서현) — 시장 검증
    ↓
Phase 2: 판교·수지 — 인접 지역 확장
    ↓
Phase 3: 일산·평촌·산본 등 1기 신도시 — 동일 모델 복제
    ↓
Phase 4: 서울 주요 상권 — 강남, 홍대, 건대 등
    ↓
Phase 5: 전국 주요 도시
```

→ 당근마켓도 판교에서 시작해 동네 단위로 확장했다. 1기 신도시들은 **고령화 + 상권 밀집 + 교통 좋음**이라는 공통점이 있어 같은 모델이 복제 가능하다.

### 8-3. 분당 런칭 시뮬레이션 (보수적 추정)

```
분당구 식당/카페/편의점 사업체:  약 4,000개
초기 가입 전환율 5%:             200개 사업체
월 평균 2건 일감 등록:           400건/월
건당 평균 일감 금액:             80,000원 (4시간 × 시급 20,000원)
수수료 15%:                      건당 12,000원
───────────────────────────────
월 거래액:                       3,200만원
월 매출 (수수료):                480만원 (초기)
```

→ 초기 매출은 작지만, 네트워크 효과 사업이다. 임계점을 넘기면 기하급수적으로 성장한다.

## 9. 리스크 & 대응 전략

### 9-1. 사업 리스크

| 리스크 | 심각도 | 대응 전략 |
|---|---|---|
| **닭과 달걀 문제** (구인 없으면 워커 안 옴) | 높음 | 분당 식당 200곳 직접 영업으로 초기 구인 확보. 워커는 시니어 커뮤니티·복지관 통해 모집 |
| **당근마켓 진출** | 높음 | 속도(즉시 호출) + 정산 + 전문 신뢰 시스템으로 차별화. 당근은 게시판, 일해요는 호출 |
| **워커 품질 불균일** | 중간 | 별점 시스템 + 노쇼 페널티 + 우수 워커 배지 + 경력 태그 자동 부여 |
| **매칭 실패** (워커 부족) | 높음 | 초기 특정 지역(분당) 집중으로 워커 밀도 확보. 넓게 퍼지면 밀도가 낮아져 매칭 실패 |

### 9-2. 법적/제도적 리스크

| 리스크 | 심각도 | 대응 전략 |
|---|---|---|
| **노동법 이슈** (4대보험, 근로계약) | 높음 | 일용직 근로계약서 자동 생성, 법적 요건 자동 준수. 법률 자문 필수 |
| **산재/사고 문제** | 높음 | 상해보험 연계 (워커 전원 자동 가입), 현장 GPS 인증으로 근무 기록 확보 |
| **세금 문제** | 중간 | 원천징수 자동 처리, 소득 신고 도우미 기능 |
| **플랫폼 노동자 보호법** | 중간 | 법률 동향 지속 모니터링, 선제적 워커 보호 정책 수립 |

### 9-3. 운영 리스크

| 리스크 | 심각도 | 대응 전략 |
|---|---|---|
| **워커 노쇼** | 높음 | 노쇼 시 평점 하락 + 일정 기간 매칭 제한. 대체 워커 자동 매칭 |
| **구인자 미정산** | 중간 | 에스크로 방식 (선입금 후 매칭). 정산 지연 시 구인자 이용 제한 |
| **분쟁 발생** | 중간 | GPS 기반 근무 기록으로 객관적 증거 확보. 분쟁 조정 프로세스 구축 |
| **시니어 디지털 리터러시** | 중간 | 극도로 단순한 UX. 최대 3탭 이내로 일감 수락. 초기 온보딩 지원 |

## 11. 영감의 원천

### 11-1. 일본 사례

EBS 다큐프라임 "면접도, 이력서도 필요 없다 선착순 N명.." 에서 다룬 일본의 현실:

- 일본은 세계에서 가장 빠르게 고령화된 나라
- **Timee(타이미)** 같은 스팟워크 앱이 폭발적으로 성장
- 70대 노인이 앱으로 480번 이상 단기 알바 수행
- 하루 평균 4시간, 면접/이력서 없이, 당일 정산
- Timee: 2024년 상장, 기업가치 약 $1B, 회원 120만+, 매출 YoY 72.6% 성장

### 11-2. 한국에서의 시사점

- 한국은 일본보다 더 빠른 속도로 고령화 진행 중
- 5~10년 내 일본과 동일한 구조로 갈 가능성 높음
- 미국도 시니어 긱워크 증가 추세 (2026년 "unretirement" 현상)
- 한국에는 아직 Timee급 지배적 플랫폼이 없음
- **"Timee 한국판 + 시니어 친화 + 양방향 신뢰 시스템"** 포지션이 비어 있음

## 12. 핵심 차별화 요약

| 기존 서비스 | 일해요 |
|---|---|
| 이력서 제출 → 면접 → 채용 | 이력서 없이 바로 투입, 경력은 플랫폼에서 쌓임 |
| 게시판에 올리고 기다림 | 우버처럼 즉시 호출·매칭 |
| 노인 = 복지 대상 | 노인 = 생산자, 경험 많은 인력 |
| 범용 알바 플랫폼 | 초단기 + 시니어 친화에 올인 |
| 정산은 알아서 | 플랫폼이 정산·계약·보험 처리 |
| 매너 온도 (범용) | 업무 특화 신뢰 시스템 (별점, 경력, 출석률) |
