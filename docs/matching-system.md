# 일해요 (1haeyo) — 매칭 시스템 설계

> 핵심 원칙: **결정 마비 방지, 양측 즉시 책임, 속도가 곧 가치**

---

## 1. 두 가지 매칭 모드

일감의 **시작 시각까지 남은 시간**에 따라 자동으로 모드가 나뉜다.

| 모드 | 조건 | 매칭 방식 | 벤치마크 |
|---|---|---|---|
| **Mode 1: 즉시 호출** | 3시간 이내 시작 | 완전 선착순 자동 매칭 | Uber, Timee, 대리운전 |
| **Mode 2: 예약 구인** | 3시간 이상 여유 | 선착순이되 구인자 판정 | 일해요 고유 설계 |

구인자는 Mode 2에서도 "즉시 매칭 원함"을 선택해 Mode 1로 강제 전환 가능.

---

## 2. Mode 1: 즉시 호출 (Uber/Timee 방식)

### 2-1. 사용 시나리오

- "알바 갑자기 안 나왔다, 지금 당장 1명 필요"
- "단체 손님 왔다, 2시간 내 홀서빙 1명"
- "오늘 저녁 8시 이벤트, 3시간 후 스태프 2명"

### 2-2. 플로우

```
[구인자]
  일감 등록 (시작 시각, 장소, 업무, 시급, 인원)
    ↓ 즉시
[시스템]
  반경 내 조건 충족 워커에게 푸시 알림 브로드캐스트
    ↓
[워커들]
  푸시 확인 → "지원" 탭
    ↓
[시스템]
  첫 번째 지원자 자동 확정 (인원만큼)
  나머지에게 "마감됨" 알림
    ↓
[확정 워커]
  일터로 출발 → GPS 추적 시작
    ↓
[구인자]
  "OO님이 오고 있어요. 예상 도착 XX시 XX분" 실시간 표시
    ↓
[현장 도착]
  워커 GPS 체크인 → 구인자 확인
    ↓
[근무 완료]
  워커 GPS 체크아웃 → 구인자 확인
    ↓
[양방향 평가]
  별점 + 리뷰
    ↓
[정산]
  에스크로 → 워커 계좌로 송금
```

### 2-3. 핵심 특징

- **선택 없음, 자동 확정**: 조건 맞으면 선착순
- **워커 입장 동기**: "빨리 지원해야 내가 잡는다"
- **구인자 입장 안정**: "등록하면 무조건 사람 온다"

### 2-4. Mode 1에서의 워커 품질 관리

구인자가 선택을 못 하니까 시스템이 품질을 보장해야 한다:

- **조건 필터링**: 시급, 경력 태그, 최소 별점, 최대 이동거리 등을 구인자가 설정
- **매칭 제외 기능**: 구인자는 특정 워커 "차단" 가능 (이 사람한테는 알림 안 감)
- **우선 호출 기능**: 이전에 좋았던 워커를 "지정 호출" 가능 (Timee에는 없는 기능)

## 3. Mode 2: 예약 구인 (일해요 고유 설계)

### 3-1. 사용 시나리오

- "내일 점심 홀서빙 1명 구함"
- "다음 주 토요일 행사 도우미 3명"
- "이번 달 말 창고 정리 2명"

### 3-2. 핵심 원칙

> **"들어오는 순서대로 즉시 판정, 모아두고 비교 불가"**

이게 왜 중요한지:

- 지원자 모아놓고 비교하는 순간 → 결정 마비
- 구인자 "더 좋은 사람 올 수도 있는데..." → 하루 종일 미결
- 워커 "여기도 지원, 저기도 지원" → 양다리 → 실제 확정 시점에 혼란
- 기존 알바 플랫폼 (알바몬, 당근)이랑 똑같아짐

### 3-3. 플로우

```
[구인자]
  예약 일감 등록
    ↓
[시스템]
  공고 게시 → 관심 워커들에게 알림
    ↓
[워커 A가 가장 먼저 지원]
    ↓
[구인자에게 푸시]
  "워커 A가 지원했습니다. 프로필 보고 판정해주세요"
  (제한 시간: 10분)
    ↓
[구인자 판정]
  ├── 【채용】
  │     → 매칭 확정
  │     → 공고 즉시 마감
  │     → 워커 A의 다른 지원건 자동 취소
  │     → 끝
  │
  └── 【거부】
        → 공고 유지
        → 워커 A는 이 공고에 재지원 불가 (3시간 잠김)
        → 다음 지원자 대기
           ↓
     [워커 B 지원]
        → 구인자 푸시 → 판정 → ...
```

### 3-4. 동시 지원 룰 (Q2 수정 반영)

**워커는 여러 일감에 동시 지원 가능.**

```
워커 X가 일감 P, Q, R 모두에 지원
    ↓
P의 구인자가 먼저 채용 확정
    ↓
시스템 자동 처리:
  - P: 매칭 확정
  - Q, R: 워커 X의 지원 자동 취소
  - Q, R의 구인자에게 "지원자 X가 다른 곳 확정되었습니다" 알림
    → Q, R은 다음 지원자 대기
```

**왜 이게 좋은가:**

1. **워커의 기회 극대화**: 한 곳만 기다리며 묶일 필요 없음
2. **구인자의 속도 동기부여**: "빨리 결정 안 하면 이 사람 놓친다" → 결정 속도 ↑
3. **당근/알바몬과 차별**: "모아놓고 비교" 금지, 즉시 결정 강제
4. **자연스러운 시장 메커니즘**: 좋은 워커는 빨리 뽑히고, 좋은 일감은 빨리 찬다

### 3-5. 구인자 판정 시간 제한

| 상황 | 시간 | 초과 시 |
|---|---|---|
| 예약 일감 (3시간 이상 여유) | **10분** | 자동 거부 처리. 워커 알림 |
| 반복 시간 초과 (신뢰도 하락) | - | 구인자 계정 경고, 지속 시 공고 제한 |

**이유**: 워커가 30분째 기다리게 하면 안 됨. 10분이 양쪽에 공평.

### 3-6. 거부된 워커 재지원 제한

- 같은 공고에 3시간 동안 재지원 불가
- 다른 공고에는 즉시 지원 가능

**이유**: 거부당한 워커가 바로 또 지원하면 구인자도 피곤하고, 구인자의 판단 가치가 떨어진다.

## 4. 상태 머신

### 4-1. 일감(Job) 상태

```
[DRAFT] → [OPEN] → [MATCHING] → [CONFIRMED] → [IN_PROGRESS] → [COMPLETED]
                ↘                                            ↗
                  [CANCELLED] ← ────────────────────────────
```

| 상태 | 의미 |
|---|---|
| DRAFT | 구인자가 작성 중 |
| OPEN | 공고 게시됨, 지원 대기 |
| MATCHING | 지원자 있음, 구인자 판정 중 (Mode 2) |
| CONFIRMED | 워커 확정, 출근 대기 |
| IN_PROGRESS | 워커 출근, 근무 중 |
| COMPLETED | 근무 완료, 정산 완료 |
| CANCELLED | 취소됨 (누가 취소했는지 기록) |

### 4-2. 지원(Application) 상태

```
[PENDING] → [ACCEPTED] → (Job은 CONFIRMED로)
         ↘ [REJECTED]
         ↘ [AUTO_CANCELLED]  ← 다른 곳에서 확정돼서 자동 취소
         ↘ [EXPIRED]          ← 구인자 판정 시간 초과
```

## 5. 페널티 시스템 (Timee 벤치마크 + 개선)

Timee가 2024년 10월 일본 정부 개입으로 페널티 수위를 낮춘 사례가 있다. 너무 엄격하면 워커가 이탈하고, 너무 느슨하면 구인자가 떠난다. 균형이 중요.

### 5-1. 워커 페널티 (포인트제)

| 상황 | 포인트 | Timee 참고 |
|---|---|---|
| 근무 **72시간 이내** 취소 | **1 pt** | Timee: 1 pt |
| 근무 **24시간 이내** 취소 | **2 pt** | Timee: 2 pt |
| 근무 **1시간 이내** 취소 | **3 pt** | Timee보다 세분화 |
| **지각** (30분 이상) | **2 pt** | Timee: 2 pt |
| **무단결근 (노쇼)** | **8 pt** | Timee: 8 pt (즉시 정지) |

**제재 단계**:
- 1~3 pt: 제한 없음
- 4~7 pt: 주 1건만 지원 가능
- 8 pt 도달: **14일 정지** + 복귀 시 4 pt부터 시작

**포인트 감소**:
- 근무 완료 후 리뷰 작성 시 **1 pt 감소**
- 무사고 근무 10회마다 추가 **1 pt 감소** (자정 기준)

**면제 조건**:
- 48시간 전 취소는 페널티 없음
- 자연재해, 응급의료 등 불가항력은 심사 후 면제

### 5-2. 구인자 페널티

일본에서 Timee가 한동안 구인자 페널티가 너무 약해서 문제됐음. 2025년 9월부터 24시간 전 취소 시 휴업수당 법적 의무화. 우리도 비슷하게 설계.

| 상황 | 조치 |
|---|---|
| 근무 24시간 이내 취소 | **시급 × 근무시간의 50%** 워커에게 지급 (휴업수당) |
| 근무 1시간 이내 취소 | **시급 × 근무시간의 100%** 워커에게 지급 |
| 워커 도착 후 돌려보냄 | **100% 전액** 지급 + 경고 |
| 임금 미지급 | 계정 정지 + 법적 대응 |
| 근무 조건 허위 기재 | 경고 → 반복 시 정지 |
| 판정 시간 초과 반복 (Mode 2) | 경고 → 공고 게시 제한 |

### 5-3. 공정성 장치

- 워커가 페널티에 이의 제기 가능 (자연재해, 질병 등)
- 구인자의 부당 거부 신고 기능
- AI 기반 이상 패턴 감지 (차별적 거부, 반복 노쇼 등)

## 6. 취소 플로우 상세

### 6-1. 워커 취소

```
[매칭 확정 후 취소 버튼 탭]
    ↓
[취소 사유 선택]
  - 개인 사정
  - 몸이 안 좋아요
  - 실수로 지원했어요
  - 기타
    ↓
[페널티 안내 화면]
  "현재 취소 시 X 포인트가 부과됩니다. 정말 취소하시겠습니까?"
    ↓
[확인]
    ↓
[시스템]
  - 구인자에게 즉시 푸시 "워커가 취소했습니다"
  - Mode 1 (즉시 호출): 자동으로 대기 중인 다음 워커에게 호출 재전송
  - Mode 2 (예약 구인): 공고 다시 OPEN 상태로. 새 지원자 받음
  - 워커 페널티 포인트 적용
  - 3시간 지원 제한
```

### 6-2. 구인자 취소

```
[취소 버튼 탭]
    ↓
[취소 사유 + 휴업수당 안내]
  "근무 시작 X시간 전입니다. 워커에게 시급의 Y%를 지급하셔야 합니다."
    ↓
[확인 + 결제]
    ↓
[시스템]
  - 워커에게 즉시 푸시 + 휴업수당 지급
  - 구인자 계정 취소 이력 기록
  - 반복 시 경고/제한
```

### 6-3. 상호 합의 취소

양쪽 다 취소에 동의하는 경우 (예: 사정이 생긴 경우):
- 페널티 없음
- 휴업수당 없음
- "상호 합의 취소" 이력만 남음 (중립)

## 7. 평가 & 신뢰 시스템

### 7-1. 양방향 평가 (Timee 벤치마크)

근무 완료 후 **양쪽 모두 평가 작성**:

**구인자 → 워커**:
- 별점 (1~5)
- 카테고리별 평가: 근태 / 업무 역량 / 태도
- 코멘트 (선택)
- **스킬 뱃지 인증**: "홀서빙 능숙", "청소 꼼꼼" 등

**워커 → 구인자**:
- 별점 (1~5)
- 카테고리별 평가: 업무 명확도 / 직원 친절도 / 환경
- 코멘트 (선택)
- 다른 워커들이 지원 전 열람 가능

### 7-2. 스킬 뱃지 시스템 (Timee 차용)

| 뱃지 | 획득 조건 |
|---|---|
| 🏃 홀서빙 마스터 | 홀서빙 업무 20회 + 평균 별점 4.5+ |
| 🧹 청소 전문가 | 청소 업무 15회 + 평균 별점 4.5+ |
| 📦 물류 숙련자 | 물류 업무 15회 + 평균 별점 4.5+ |
| ⏰ 정시 출근왕 | 연속 30회 정시 출근 |
| 🌟 우수 워커 | 종합 별점 4.8+, 완료 50회+ |

→ 뱃지는 이력서를 대체하는 **시각적 신뢰 신호**

## 8. GPS 추적 & 출퇴근

배민/쿠팡이츠/대리운전 앱과 동일한 수준의 백그라운드 GPS 추적 구현. 앱이 백그라운드에 있거나 화면이 꺼진 상태에서도 지속 추적 가능.

### 8-1. 기술 구현

- **Android**: Foreground Service + 지속 알림 ("일해요 - 일터로 이동 중")
  - OS가 앱을 쉽게 죽이지 못하게 Foreground Service 등록
  - 사용자가 의도적으로 강제 종료하지 않는 한 계속 작동
- **iOS**: `UIBackgroundModes: location` + "항상 허용" 권한
  - 화면 꺼진 상태, 앱 백그라운드에서도 추적 계속
  - Significant Location Change API로 강제 종료 후에도 재개 가능
- **구현 라이브러리**:
  - MVP: `expo-location` (무료, 배민 수준 가능)
  - 스케일업: `react-native-background-geolocation` ($899 앱당 일회성, Uber/Lyft 급)

### 8-2. 추적 시작 시점

**Mode 1 (즉시 호출)**: 매칭 확정 시 즉시 GPS 추적 시작 (구인자에게 실시간 위치)
**Mode 2 (예약 구인)**: 근무 시작 1시간 전부터 GPS 추적 시작

### 8-3. 추적 플로우

```
[매칭 확정]
  → Foreground Service 시작
  → "일해요 - 일터로 이동 중" 알림 표시 (사용자가 앱 안 닫게)
  → GPS 위치 30초마다 서버 전송
  → Supabase Realtime으로 구인자 화면에 실시간 표시

[일터 50m 진입 - 지오펜싱]
  → 자동 체크인 제안 알림
  → 워커 확인 시 체크인 완료
  → GPS 추적 중단 (배터리 절약)

[근무 완료 체크아웃]
  → Foreground Service 완전 종료
```

### 8-4. 배터리 최적화

- `Accuracy.Balanced` 사용 (Highest 대비 70% 절감)
- 정지 상태 감지 시 GPS 빈도 감소 (motion detection)
- `deferredUpdatesDistance` 설정 (일정 거리 이동 시에만 업데이트)

### 8-5. 프라이버시 & 권한 UX

- 추적은 근무 관련 시간대에만 작동 (매칭 확정 → 체크아웃)
- 근무 중에는 추적 중단 (배터리 + 프라이버시)
- 데이터는 근무 완료 후 30일 보관 → 자동 삭제

**권한 요청 UX (시니어 친화)**:

## 9. 엣지 케이스 처리

### 9-1. 동시 지원 폭주 (Mode 2)

```
워커 A, B, C가 0.5초 간격으로 지원
    ↓
시스템 FIFO 큐에 저장
    ↓
구인자에게 A 먼저 표시 (10분 타이머)
B, C는 "대기 중 2번째, 3번째"로 워커에게 표시
    ↓
구인자가 A 거부 → B 즉시 표시
또는 A 채용 → 공고 마감, B/C에게 "다른 워커 확정" 알림
```

### 9-2. 매칭 후 연락 두절

- 근무 시작 30분 전 GPS 미감지 → 워커에게 푸시 + SMS
- 근무 시작 시각 도달 → 구인자에게 "긴급 상황" 알림
- Mode 1 자동 재매칭 옵션 제공 (반경 내 다른 워커에게 긴급 호출)

### 9-3. 근무 중 분쟁

- 앱 내 "도움 요청" 버튼 (양쪽)
- 플랫폼 상담원 개입 (초기에는 창업자 직접)
- GPS 기록 + 체크인/아웃 기록이 객관적 증거

### 9-4. 자연재해 / 불가항력

- 정부 발표 기상특보, 교통 마비 등은 시스템이 자동 감지
- 해당 시간대 매칭된 건은 양쪽 페널티 면제
- 취소 시 "자동 면제" 처리

### 9-5. 미성년자 / 외국인 등 예외

- 가입 시 주민번호 인증 (만 18세 이상 확인)
- 외국인: 체류자격 확인 (E-9, F-계열 등)
- 이러한 제약은 초기에는 단순화, 점진적 도입

## 10. Timee 대비 우리 설계의 차이

| 항목 | Timee | 일해요 |
|---|---|---|
| 매칭 방식 | 100% 자동 선착순 | 긴급=자동 선착순, 예약=구인자 판정 |
| 구인자 선택권 | 없음 | 예약 구인에서는 있음 |
| 워커 동시 지원 | 1건만 | 여러 건 가능, 확정 시 나머지 자동 취소 |
| 판정 시간 제한 | N/A | 10분 (구인자 판정) |
| 거부된 워커 재지원 | N/A | 같은 공고 3시간 잠금 |
| 스킬 뱃지 | 있음 | 있음 (차용) |
| 양방향 리뷰 | 있음 | 있음 (차용) |
| 페널티 포인트제 | 있음 | 있음 (차용, 더 세분화) |
| 휴업수당 (구인자) | 2025.9~ 의무 | 처음부터 시스템화 |
| GPS 추적 | 체크인/아웃만 | 실시간 (Uber 방식) |
| 우선 호출 (단골) | 없음 | 있음 |

## 12. 핵심 원칙 요약

1. **결정 마비 금지** — 들어오는 순서대로 즉시 판정
2. **모아놓고 비교 금지** — 기존 알바 플랫폼과의 결정적 차별점
3. **양측 책임 즉시** — 취소 = 페널티 (공정한 수준)
4. **속도가 가치** — 빠른 결정이 좋은 매칭을 만든다
5. **신뢰는 플랫폼이** — 이력서 대신 별점, 뱃지, 경력 데이터
6. **투명한 룰** — 페널티/면제 기준 공개, 이의제기 가능
