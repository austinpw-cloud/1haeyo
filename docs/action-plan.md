# 일해요 (1haeyo) — 현실적 액션 플랜

> 자본 최소, 1인 창업 전제. 냉정한 시각으로.

---

## 1. 지역 선정 분석

### 1-1. 일해요의 이상적 지역 조건

일해요가 작동하려면 **두 가지가 동시에 높은 밀도로 존재**해야 한다:
- **워커 공급**: 시니어(55~75세) + 유휴 인력이 많은 곳
- **구인 수요**: 식당, 카페, 편의점, 소매 등 단기 인력이 필요한 사업체가 밀집한 곳

추가 조건:
- 도보/대중교통 접근성 (하이퍼로컬이니까)
- 지역 커뮤니티가 활발 (입소문 가능)

### 1-2. 후보 지역 비교

#### 수도권 1기 신도시 (1991~1996 입주)

| 지역 | 전체 인구 | 65세+ 추정 | 노령화지수 | 사업체 밀집 | 교통 | 종합 |
|---|---|---|---|---|---|---|
| **분당구** (성남) | ~47만 | ~8만+ | 120.7 | 높음 (미금·야탑·서현·정자 상권) | 신분당선+분당선 | ★★★★★ |
| **일산동구** (고양) | ~30만 | ~4.9만 (16.5%) | 중간 | 중간 (정발산·마두 상권) | 3호선 | ★★★☆☆ |
| **일산서구** (고양) | ~28만 | ~4.8만 (16.9%) | 중간 | 중간 | 3호선 | ★★★☆☆ |
| **평촌** (안양 동안구) | ~28만 | 높음 추정 | 높음 (노후 93.1%) | 중간 (평촌역 상권) | 4호선 | ★★★★☆ |
| **산본** (군포) | ~18만 | 높음 추정 | 높음 (노후 88.2%) | 낮음~중간 | 4호선 | ★★★☆☆ |

#### 서울 고령인구 비율 상위 자치구

| 지역 | 65세+ 비율 | 사업체 밀집 | 시니어 소득 수준 | 종합 평가 |
|---|---|---|---|---|
| **강북구** | 23.2% (서울 1위) | 낮음 | 낮음 | ★★☆☆☆ (수요 부족) |
| **도봉구** | 22.2% | 낮음 | 낮음 | ★★☆☆☆ |
| **중구** | 20.7% | 높음 | 혼재 | ★★★☆☆ |
| **강남구** | 중간 | 매우 높음 | 높음 | ★★★★☆ (경쟁 심함) |

### 1-3. 결론: 분당이 최적인 이유

분당은 다른 후보 대비 **모든 조건을 동시에 충족**하는 거의 유일한 지역이다:

1. **시니어 밀도**: 1991년 입주 → 30대에 입주한 사람이 지금 65세+. 고소득 은퇴 전문직 다수
2. **상권 밀도**: 미금역·야탑역·서현역·정자역 4개 역세권 상권이 걸어서 닿는 거리에 밀집
3. **사업체 수**: 성남시 전체 66,333개 중 분당이 약 35~40% 차지 추정
4. **교통**: 신분당선+분당선으로 역세권 도보 생활권
5. **시니어 수준**: 분당 시니어는 지적·자산적 수준이 높고, 디지털 적응도 양호
6. **커뮤니티**: 30년 동네 → 아파트 커뮤니티, 경로당, 복지관 등 입소문 채널 존재

**강북구·도봉구**는 고령인구 비율은 높지만 상권이 약하고 시니어 소득이 낮아서 구인 수요가 부족.
**강남구**는 상권은 강하지만 경쟁이 심하고, 시니어보다 청년 알바 공급이 이미 충분.

### 1-4. 분당 내 최적 시작점

**미금역 ~ 야탑역 구간** (반경 2km)

이유:
- 미금역: 먹자골목 + 상가 밀집. 식당·카페 수백 개
- 야탑역: 대형 상권 + 주거 밀집. 시니어 거주 아파트 단지 다수
- 두 역 사이 도보 15분. 하이퍼로컬 테스트에 완벽한 사이즈
- 서현역·정자역은 2차 확장 지역

## 2. 구인자(사업체) 현실 분석

### 2-1. 분당 미금~야탑 구간 구인 수요 추정

| 업종 | 추정 사업체 수 | 단기 인력 필요 빈도 | 우선순위 |
|---|---|---|---|
| **식당** (한식, 중식, 일식 등) | 300~500개 | 높음 (주말 피크, 단체, 노쇼) | ★★★★★ |
| **카페/베이커리** | 100~200개 | 중간 (주말, 시즌) | ★★★★☆ |
| **편의점** | 50~80개 | 중간 (야간, 주말 대타) | ★★★☆☆ |
| **학원** | 100~200개 | 낮음~중간 | ★★☆☆☆ |
| **부동산** | 50~100개 | 낮음 | ★★☆☆☆ |
| **병원/약국** | 30~50개 | 낮음 | ★☆☆☆☆ |

→ **MVP 타겟: 식당 + 카페 = 400~700개 사업체**
→ 이 중 5% 전환 = **20~35개 활성 구인 사업체** (현실적 초기 목표)

### 2-2. 구인자의 진짜 Pain Point

식당/카페 사장님 입장에서 생각해보면:

```
"알바가 갑자기 안 나와요" → 가장 급한 상황. 점심 피크 전에 해결해야 함
"주말만 사람이 필요해요" → 정규 채용은 부담, 주말만 오는 사람 구하기 어려움
"단체 예약이 잡혔어요" → 하루만 추가 인력이 필요
"알바 구하기 자체가 어려워요" → 만성 인력 부족
```

→ **"알바 노쇼 긴급 대타"가 가장 강력한 초기 훅(hook)**
→ 이 한 가지 use case만으로도 가치를 증명할 수 있다

### 2-3. 구인자가 일해요를 안 쓰는 이유 (장벽)

| 장벽 | 심각도 | 대응 |
|---|---|---|
| "워커가 몇 명이나 있는데요?" | 치명적 | 초기 워커 50명 이상 확보 후 영업 시작 |
| "검증 안 된 사람 보내면 어쩌죠" | 높음 | 신분 인증 + 워커 프로필(경력, 별점) 투명 공개 |
| "앱 깔기 귀찮아요" | 중간 | 카카오톡 채널로 대체 가능 (MVP) |
| "돈은 어떻게 줘요" | 중간 | 초기: 현장 직접 정산 / 정산 시스템 붙이면 편의성 어필 |

## 3. 정산 시스템 — 처음부터 가능하다

### 3-1. 정산은 생각보다 어렵지 않다

| 항목 | 현실 |
|---|---|
| **PG사 가입비** | 포트원(PortOne) 추천패키지 → **무료** (22만원 지원) |
| **PG사 수수료** | 카드 결제 기준 약 3.4% |
| **연동 기간** | 포트원 SDK 사용 시 **2주 내 완료** |
| **정산 주기** | D+3~7 영업일 |
| **필요 기술** | 포트원 SDK + 카카오페이/토스페이 연동 |

### 3-2. 정산 플로우

```
1. 구인자가 일감 등록 시 시급 × 시간 = 총액 입력
2. 워커 매칭 확정 시 → 구인자 카드/카카오페이로 선결제 (에스크로)
3. 워커 근무 완료 + 구인자 확인
4. 플랫폼이 총액에서 수수료(10~15%) 차감 후 워커에게 정산
5. 정산은 D+3~7일 이내 워커 계좌로 입금
```

### 3-3. 왜 처음부터 붙여야 하나

- **수익이 바로 발생**: 정산 없으면 수익 0. 있으면 첫 거래부터 수수료 수입
- **구인자 편의**: "현금 안 챙겨도 됨, 세금 처리 자동" → 오히려 영업 포인트
- **신뢰**: 플랫폼이 돈을 관리 → 워커도 안심 (떼일 걱정 없음)
- **데이터**: 거래 기록 = 사업 실적 데이터 = 투자 유치에 핵심

### 3-4. 정산 관련 법적 고려

| 항목 | 내용 |
|---|---|
| 일용직 원천징수 | 일당 15만원 초과 시 원천징수 의무 (세율 약 6%) |
| 근로계약서 | 1일 단위라도 근로계약서 필요 → 앱에서 자동 생성 |
| 산재보험 | 1일 이상 근로자도 산재보험 적용 대상 |
| 사업자등록 | 플랫폼 운영 시 통신판매업 신고 필요 |

→ 법적 이슈는 복잡하지만 초기 MVP에서는 "중개 플랫폼"으로 시작하고, 스케일업 시 법률 자문을 받는 것이 현실적.

## 4. 서비스 확장 — "할 일이 있는 삶"

### 4-1. 원해요 (1haeyo) 브랜드 확장

"1haeyo = 원해요 + 일해요"라는 브랜드가 강력한 이유:

돈을 버는 일만이 아니라, **원하는 활동을 연결**하는 플랫폼으로 확장 가능.

| 카테고리 | 예시 | 보상 |
|---|---|---|
| **일해요 (유급)** | 식당 알바, 사무보조, 물류 | 시급 |
| **원해요 (무급/봉사)** | 동네 청소, 독거노인 말벗, 도서관 보조 | 활동 포인트, 사회적 인정 |
| **배워요** | 스마트폰 교실, 요리 교실, 운동 모임 | 커뮤니티 |
| **나눠요** | 재능 기부, 멘토링, 경험 공유 | 보람 + 평판 |

→ 시니어의 핵심 니즈: **"돈"만이 아니라 "할 일이 있는 삶", "사회적 연결", "인정받는 느낌"**
→ 일해요가 54.4%의 "생활비 보탬" 니즈를 잡고, 원해요가 36.1%의 "일하는 즐거움" + 4%의 "무료해서" 니즈를 잡는다

### 4-2. 이게 사업적으로도 의미 있는 이유

- **워커 리텐션**: 돈 안 되는 날에도 앱을 켜게 만든다
- **커뮤니티 락인**: 동네 활동 → 관계 형성 → 앱 이탈 감소
- **데이터**: 활동 이력이 워커 프로필을 풍부하게 만든다
- **마케팅**: "노인 알바앱"이 아니라 **"시니어 라이프 플랫폼"** → 언론, 지자체 관심

## 5. 홍보·마케팅·영업 방안

### 5-1. 자본 거의 없는 전제에서의 마케팅

**원칙: 돈 쓰는 마케팅 NO. 발로 뛰는 마케팅 + 입소문 + 무료 채널**

### 5-2. 워커 모집 (구직자 확보)

| 채널 | 방법 | 비용 | 효과 |
|---|---|---|---|
| **아파트 경로당/복지관** | 직접 방문. "이런 앱이 있는데 같이 해보실래요?" 설명회 | 0원 | ★★★★★ (시니어 직접 접촉 가능) |
| **네이버 밴드** | 분당 지역 시니어 밴드, 동네 밴드에 홍보글 | 0원 | ★★★★☆ (시니어가 밴드 많이 씀) |
| **카카오톡 채널** | "일해요 분당" 카카오 채널 개설, 일감 알림 발송 | 0원 | ★★★★☆ |
| **아파트 게시판** | 엘리베이터 공지, 관리사무소 협조 | 0원~소액 | ★★★☆☆ |
| **성남시 시니어 센터** | 50플러스센터, 시니어클럽 연계 | 0원 | ★★★★☆ (공공기관 신뢰) |
| **지역 신문/카페** | 분당신문, 네이버 카페 "분당맘" 등 | 0원 | ★★★☆☆ |
| **지인 네트워크** | 부모님 세대 → 부모님 친구 → 입소문 | 0원 | ★★★★★ (초기 가장 강력) |

**핵심 전략: 시니어는 "사람이 직접 와서 보여주면" 움직인다.**
→ 경로당/복지관 5곳 방문해서 10명씩 = 50명 워커 확보 가능

### 5-3. 구인자 영업 (사업체 확보)

| 채널 | 방법 | 비용 | 효과 |
|---|---|---|---|
| **미금역 먹자골목 도보 영업** | 식당 한 곳씩 방문. 명함 + 전단지 + 앱 시연 | 전단지 인쇄비 ~3만원 | ★★★★★ |
| **야탑역 상가 도보 영업** | 카페, 편의점, 소매점 방문 | 동일 | ★★★★☆ |
| **소상공인 협회/상인회** | 미금역 상인회 등에 단체 제안 | 0원 | ★★★★☆ (한 번에 다수 접촉) |
| **카카오톡 채널 (사업자용)** | "일해요 사장님" 채널로 일감 등록/매칭 알림 | 0원 | ★★★☆☆ |
| **네이버 플레이스 DM** | 미금역 주변 식당에 네이버 플레이스 통해 연락 | 0원 | ★★☆☆☆ (응답률 낮음) |

**핵심 전략: "지금 워커 50명 대기 중입니다. 무료로 써보세요."**
→ 초기 수수료 0원 프로모션으로 장벽 제거
→ 3개월 무료 후 수수료 도입

### 5-4. 영업 스크립트 (식당 사장님용)

```
"사장님, 혹시 알바 갑자기 안 나올 때 어떻게 하세요?"
(대부분: "그냥 제가 뛰죠" 또는 "급하게 전화 돌리죠")

"저희가 그거 해결해드리는 앱이에요. 
지금 이 동네에 바로 나올 수 있는 분들이 50명 대기 중이거든요.
사장님이 앱에서 '오늘 12시 홀서빙 1명' 올리시면
5분 안에 매칭돼서 사람이 와요.
3개월은 완전 무료고, 한번 써보시겠어요?"
```

### 5-5. PR/언론

| 채널 | 접근법 | 기대 효과 |
|---|---|---|
| **지역 언론** | 분당신문, 성남일보에 보도자료 | 지역 인지도 |
| **시니어 미디어** | 브라보마이라이프, 시니어매거진 | 시니어 타겟 직접 도달 |
| **스타트업 미디어** | 벤처스퀘어, 아웃스탠딩, 플래텀 | 투자자 관심 |

| **성남시청** | 시니어 일자리 정책 연계 제안 | 공공 신뢰 + 지원금 가능성 |

## 6. 냉정한 리스크 재점검

### 6-1. 치명적 리스크

| 리스크 | 현실적 시나리오 | 대응 |
|---|---|---|
| **닭과 달걀** | 워커 50명 모았는데 구인 0건. 워커 이탈 시작 | 워커 모집과 구인 영업을 동시에. 최소 10곳 사업체 확보 전에는 워커 대량 모집 자제 |
| **당근마켓** | 당근이 "당근알바"에 실시간 매칭+정산 붙이면 | 속도 싸움. 당근이 인식하기 전에 분당 장악. 데이터+커뮤니티 락인 |
| **워커 품질** | 첫 번째 워커가 일을 못 하면 → 구인자 "다신 안 써" | 초기 워커 직접 면담(비공식). 우수 워커를 먼저 매칭하는 큐레이션 |
| **법적 이슈** | 노동법 위반 소지, 산재 발생 시 책임 | 법률 자문 1회 (비용 10~30만원). 근로계약서 자동화. 보험 연계 |

### 6-2. 실패 시나리오별 손실

| 시나리오 | 투입 비용 | 투입 시간 | 손실 |
|---|---|---|---|
| MVP 만들고 영업 시작 전 포기 | ~10만원 (서버비) | 2~4주 | 거의 없음 |
| 3개월 운영 후 매칭 안 됨 | ~30만원 | 3개월 | 적음 |
| 6개월 운영 후 스케일 실패 | ~50만원 | 6개월 | 중간 (시간) |
| 1년 풀타임 투입 후 실패 | ~100만원 | 1년 | 시간 손실이 큼 |

→ **MVP + 3개월 테스트까지는 금전적 리스크 거의 없음. 핵심 리스크는 시간.**

## 10. 핵심 정리

### 한 줄 요약
> 분당 미금~야탑 반경 2km에서, 식당 알바 노쇼 긴급 대타 한 가지 문제를 풀면서 시작한다.

### 성공 공식
```
좁은 지역 (미금~야탑 2km) 
× 하나의 use case (긴급 대타)
× 소수의 사업체 (식당 20곳)
× 소수의 워커 (50명)
= 제대로 돌아가는 최소 단위
```

### 하지 않을 것
- 전국 서비스 (X) → 분당 2km만
- 모든 업종 (X) → 식당/카페만
- 완벽한 앱 (X) → 웹앱으로 핵심만
- 유료 마케팅 (X) → 발로 뛰기
- 처음부터 팀빌딩 (X) → 혼자서 검증

### 최악의 시나리오
3개월 후 안 되면? 손실은 시간 3개월 + 비용 30만원. 감당 가능.
