# 결제 / 정산 / 계약 모델

최종 확정: 2026-04-14.

## 원칙

- **에스크로 선결제**: 사장님이 일감 등록 시 총액을 에스크로로 예치해야 공고 노출.
- **비용 투명 공시**: 근무 대가 · 플랫폼 수수료 · PG 수수료를 분리해 표기하고 총액 계산.
- **양측 전자서명 → 자동 계약**: 사장님(등록 체크박스) + 워커(지원 체크박스) = 매칭 확정 순간 전자 근로계약서 자동 생성·저장·양측 고지.
- **당일 즉시 정산**: 근무 완료 + 사장님 승인 즉시 워커 계좌로 송금 (토스페이먼츠 즉시송금 API).

---

## 비용 구조 예시

시급 15,000원 × 2시간 기준:

| 항목 | 금액 | 수취자 |
|---|---|---|
| 근무 대가 (시급 × 시간) | 30,000원 | 워커 |
| 플랫폼 수수료 10% | 3,000원 | 일해요(플랫폼) |
| PG 정산 수수료 ~3.3% (토스페이먼츠) | 약 1,000원 | PG사 |
| **사장님 총 지출** | **약 34,000원** | — |

---

## 수수료율 로드맵

| 기간 | 플랫폼 수수료 |
|---|---|
| 런칭 0~3개월 | **0% 프로모션** (사장님 확보 우선) |
| 3~6개월 | **10%** (Timee · 급구 수준) |
| 6개월+ | **15%** (거래량 쌓이면 상향) |
| 긴급 매칭 (3시간 내) | **+3%** 프리미엄 |

---

## 서비스 플로우

1. 사장님 일감 등록 폼 입력 → 시급 × 시간 = 기본액 계산
2. 앱이 총액 breakdown 표시 (근무 대가 + 플랫폼 수수료 + PG 수수료)
3. **"근로계약 조건 확인하고 등록합니다"** 체크 (= 사장님 전자서명)
4. 결제 (에스크로) — 플랫폼 계좌에 예치, 공고 노출
5. 워커 지원 시 **"근로계약 조건 확인하고 지원합니다"** 체크 (= 워커 전자서명)
6. 매칭 확정 (Mode 1 자동 / Mode 2 사장님 수락) → 전자 근로계약서 자동 생성·저장
7. 양측에 "근로계약 체결됨" 고지 + 상세 조회 가능
8. 근무 → 체크인 (현장 150m 이내 GPS 검증) → 근무 → 체크아웃 + 사장님 승인
9. 에스크로에서 즉시 정산:
   - 워커 계좌로 근무 대가 즉시 송금 (토스페이먼츠 즉시송금 API)
   - 플랫폼 수수료는 플랫폼 계좌로
   - PG 수수료는 결제 시점에 자동 차감
10. 취소 규칙은 docs/matching-system.md §5 페널티 시스템(휴업수당 포함) 적용

## 근로계약서 법적 요건

근거: 근로기준법 §17 (모든 근로자, 1시간 근무 포함) + 전자문서법 §4 + 전자서명법(2020 개정).

필수 4요소 (분쟁 대비):

1. **서명자의 명확한 의사 표시** — 체크박스 + 버튼 탭 + 타임스탬프
2. **계약 내용** — 시급, 시간, 장소, 업무 등
3. **시각 기록** — 서버 타임스탬프 (ISO 8601)
4. **변조 방지** — SHA-256 해시 / 타임스탬프 / 감사 로그

### 단계별 강화

- **MVP (현재)**: `legal_status = 'simple_consent'` — 앱 내 체크박스 + 메타데이터 기록.
- **런칭 후**: `legal_status = 'certified_identity'` — PASS / 카카오 인증 / 토스 인증 동반 본인 확인.
- **정식화**: `legal_status = 'notarized'` — KISA 공인 타임스탬프 연동. 근로감독관 제출 가능한 공식 포맷.

## DB 스키마

- **contracts 테이블** (supabase/schema/008_contracts_and_payout.sql): `job_id`, `employer_id`, `worker_id`, `contract_body` (JSONB 스냅샷), `content_hash`, 양측 서명 타임스탬프/IP/기기.
- **jobs 테이블**: `platform_fee_rate`, `pg_fee_rate`, `worker_pay`, `platform_fee`, `pg_fee`, `total_amount_charged`, `escrow_status`.
- **profiles 테이블**: `bank_name`, `bank_account_number`, `bank_account_holder` (워커 정산 계좌).
- **matches 테이블**: `payment_status` (`pending` → `escrow` → `settled` → `refunded`).

## 법적 · 운영 요구사항

| 항목 | 내용 |
|---|---|
| **전자 근로계약서** | 매칭 확정 시 자동 생성·양측 서명 기록 보존 (근로기준법 제114조 위반 시 500만원 이하 벌금) |
| **산재보험** | 1일 이상 근로자도 가입 대상. 플랫폼에서 자동 처리 구조 필요 (향후 단계) |
| **원천징수** | 일당 15만원 초과 시 약 6% 원천징수 |
| **통신판매업 신고** | 플랫폼 결제 개시 전 필수 |
| **사업자등록** | 개인 or 법인. PG 계약 선결 |
| **휴업수당 (2025.9~ 의무)** | 구인자 24시간 내 취소 시 시급 × 근무시간 × 50% 워커 지급 |

## 벤치마크 근거

- **Timee (일본)**: 사장님 10% 수수료 + 당일 정산 → 연 3,000억원 매출, 2024 도쿄증시 상장, 기업가치 $1B.
- **급구 (한국)**: ~15% 수수료 + "당일 급여" 마케팅 → 분기별 5배 성장.
- **20% 이상**: 사장님 저항 심해 초기 시장 확장 저해. 10~15% 구간이 표준.
